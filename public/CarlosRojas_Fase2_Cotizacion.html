Resumen

Implementación “backend primero” de modo invitado (límite 5 sesiones), Login con Google (OAuth), i18n es/en en API y prompts, asistente más social, CORS/seguridad y contratos API unificados, y ajustes de frontend (HUD, Enter, atajos, responsive). Proyecto en curso (inicio 24/08/2025). Pago único.

Alcance

  • Invitado (hasta 5 sesiones): token/cookie guest, contador atómico y gate con code:"GUEST_LIMIT".
  • Login con Google (OAuth): flujo state/nonce/PKCE, upsert de usuario y emisión de JWT propio.
  • i18n (es/en) en API y prompts: Accept-Language / ?lang, preferred_language, plantillas localizadas.
  • Asistente más social: tono coach amigable, seguro; contexto (texto, WPM, score, errores); reintentos y timeouts.
  • Contratos API y errores: createRsvp, createQuiz, assistant, StatsResponse; errores { code, detail }.
  • CORS/Seguridad: orígenes permitidos (dev/prod), credenciales si cookie; SameSite/Secure en prod.
  • Ajustes frontend : HUD/Enter/shortcuts/responsive; callback OAuth; respeto de GUEST_LIMIT.
  • Pruebas/verificación: colecciones Postman/Bruno, unit (Zod/JSON Schema) e integración básica.
