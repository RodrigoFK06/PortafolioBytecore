1) Contexto y objetivos

Cierre de ventas en WhatsApp con pago previo y validación humana previa a la entrega IA (A: texto / B: PDF/Excel). Seguimiento 3–7 días para elevar conversión y NPS. Redis mantiene coherencia (4–5 mensajes) y rate‑limit evita spam.

Diagrama ASCII

Usuario WA → Menú (FAQs | Precios | Asesorías)
                      │           │
                      └→ IA FAQ   └→ Pago (QR/Link/JPEG) → Notifica dueño → Confirma
                                                      └→ Entrega IA (Texto | PDF/Excel)
n8n (flujos, colas, reintentos) → Redis (contexto 4–5 / rate-limit)
                              → LLM (GPT-4o mini) → Storage S3
                              → PostgreSQL (métricas/ventas/estado)
                              → Seguimiento 3–7 días (tips/upsell/NPS)

2) Alcance funcional (resumen)

  • Menú FAQs/Contexto (búsqueda en corpus), Precios/Promos, Asesorías/Planes con pago previo.
  • Validación humana (WhatsApp del dueño: Aprobar/Rechazar) y registro en PostgreSQL.
  • Entrega: Plan A texto; Plan B PDF/Excel con link firmado (expiración).
  • Seguimiento: drip 3–7 días + encuesta NPS.
  • Guardrails: lista de contenido sensible, cooldown, bloqueo temporal.

3) Arquitectura técnica

  • WhatsApp Business Cloud API (Meta) con webhooks y plantillas aprobadas.
  • n8n (orquestación, ramas de aprobación, reintentos).
  • Redis (memoria corta, rate‑limit).
  • PostgreSQL (Render) obligatorio para trazabilidad, ventas y auditoría.
  • Storage S3 para medios/PDF; URLs firmadas.
  • LLM GPT‑4o mini; parámetros conservadores y cache de prompts cuando aplique.
  • (Plan B) ASR para voz→texto; generador PDF/Excel.
  • Seguridad: HMAC webhooks, RBAC, rotación de secretos, logs estructurados.
  • Métricas: panel de éxito/fallo, tiempos y aprobaciones.

4) Planes y entregables

Plan A – Básico (solo texto)

  • Incluye: FAQs • Precios/Promos • Asesorías con pago previo • Validación humana • Entrega texto • Seguimiento 3–7 días • Redis • Postgres • Métricas base
  • Límites: 3 menús; corpus inicial ≤ 200 MB; 5 plantillas WA.
  • SLA: 8×5; P1 < 8 h; P2 1 día; P3 2–3 días.
  • No incluye: voz/imágenes, PDFs/Excels, integraciones ERP, BI avanzado.

Plan B – Pro (voz/imagen + PDF/Excel)

  • Incluye todo lo de A + ASR (voz→texto), lectura básica de imágenes y generación de PDF/Excel con branding.
  • Límites: corpus ≤ 500 MB; 10 plantillas; 1 h de audio/mes incluido (estimación).
  • SLA: 12×5; P1 < 6 h; P2 < 12 h.
  • No incluye: integraciones contables/ERP, multicliente, BI avanzado.

8) Supuestos y exclusiones

  • Cliente provee corpus y valida políticas WA.
  • Uso de plantillas conformes a Meta para mensajes proactivos.
  • Límite de archivos/voz según plan; sin traducciones ni etiquetado manual.
  • No incluye ERP/CRM legado ni BI; multicliente se cotiza aparte.

9) RACI & criterios de aceptación

  • Cliente: A/R en objetivos, corpus, accesos WABA/pagos; R en validación de pagos.
  • Bytecore: A/R en diseño conversacional, flujos, n8n/Redis/Postgres/LLM, observabilidad.
  • Aceptación: pago previo obligatorio; validación humana efectiva; entrega (texto / PDF/Excel); métricas de tiempo de respuesta y tasa de entrega operativas.

10) Riesgos y mitigaciones

  • Bloqueos WA → plantillas aprobadas, opt‑in, monitoreo.
  • Picos → colas/reintentos, rate‑limit, backoff.
  • Caídas de terceros → circuit‑breakers, reintentos y fallback informativo.
  • Contenido sensible → guardrails y listas de bloqueo.
